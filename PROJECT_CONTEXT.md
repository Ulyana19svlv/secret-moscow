# Контекст проекта Family Hub

Этот файл - живая заметка для Codex и Ульяны. Его нужно обновлять после важных изменений, чтобы при сжатии контекста не терять решения, ссылки, структуру и текущий статус проекта.

## Что это

`family-hub` - семейный портал Маши с несколькими базами:

- `Секретная Москва` / места: рабочая база мест, куда хочется сходить.
- `Медицина семьи`: подключается отдельной ссылкой/репозиторием, документы не переносим без явного разрешения.
- `Дом и быт`: пока будущий раздел.

Главная идея: один вход для разных семейных баз, но каждая база может жить как отдельное приложение или отдельный репозиторий, если так безопаснее.

## Формат места в `site/places.js`

Каждое место - объект примерно такого вида:

```js
{
  id: "unique-id",
  title: "Название",
  category: "рестораны и кафе",
  city: "Москва",
  address: "Адрес",
  coords: [55.7558, 37.6176],
  description: "Короткое описание.",
  links: {
    instagram: "https://www.instagram.com/...",
    telegram: "https://t.me/...",
    site: "https://..."
  },
  media: {
    type: "instagram",
    url: "https://www.instagram.com/reel/...",
    label: "Reels из Instagram",
    icon: "instagram"
  },
  tags: ["тег", "тег"],
  access: {
    fromHome: {
      km: 7.1,
      minutes: 25,
      note: "примерно, без учета пробок"
    },
    metro: [
      { name: "Станция", distanceKm: 0.4 }
    ]
  }
}
```

Если место не из Москвы:

- все равно добавлять в общий `site/places.js`;
- обязательно указать `city`;
- в `access.fromHome.note` честно писать, что расчет примерный из Москвы или что локация меняется;
- если адреса нет, можно поставить примерные координаты города/центра и явно написать в `address`, что точная локация назначается заранее.

## Дизайн-направление

Желаемое настроение: дофаминовый, редакционный гид, яркий и стильный, без колхоза.

Референсы от Ульяны:

- акцентные красивые шрифты;
- яркие, но собранные сочетания цветов;
- абстрактные формы, паттерны, наклейки, живые иллюстративные элементы;
- карточки должны ощущаться как маленькие редакционные материалы, а не сухая таблица.

При этом важно:

- не ломать читаемость;
- не обрезать текст;
- на мобильном список и карта должны быть управляемыми;
- карта сейчас должна быть заметно меньше по роли, чем контент и карточки;
- в карточках нужно предусматривать медиа: картинку, видео или Instagram/Reels.

## Яндекс.Карты

- Используется Yandex Maps JS API.
- Публичный ключ сейчас встроен в `site/app.js` как `DEFAULT_YANDEX_MAPS_KEY`.
- Если карта просит ключ на телефоне, проверять:
  - HTTP Referer в кабинете Яндекса;
  - кэш страницы и cache-bust `?v=...`;
  - что в HTML подключается свежий `site/app.js?v=...`.

## Безопасность и важные правила

- Не коммитить Trello token, приватные медицинские данные, личные документы, ключи и пароли.
- Медицинская база важная: не переносить, не удалять и не реорганизовывать ее без прямого запроса.
- Если пользователь говорит, что медицинскую заметку/файл удалить, сначала проверить репозиторий/статус, затем удалить, закоммитить и запушить только нужное.
- Уже было: заметку `D Артём.md` в медицинском репозитории пользователь просила удалить везде и закоммитить.

## Как добавлять новые места

Обычно пользователь присылает пачку ссылок из Instagram/Telegram и адрес/категорию. Нужно:

1. Добавить место в `site/places.js`.
2. Найти/уточнить координаты по адресу.
3. Добавить `city`.
4. Добавить `category`.
5. Добавить `links`: сайт, Instagram, Telegram, если есть.
6. Добавить `media`, если есть reels/фото.
7. Добавить `access.fromHome`: приблизительно от адреса `Дохтуровский переулок, дом 6`.
8. Добавить ближайшие станции метро в `access.metro`.
9. Обновить счетчик мест в `index.html`.
10. Повысить cache-bust версию в `index.html` и `site/index.html`.
11. Прогнать проверки.
12. Commit + push.

Минимальные проверки:

```powershell
node --check site/app.js
node -e "const fs=require('fs'); const p=JSON.parse(fs.readFileSync('site/places.js','utf8').replace(/^window\.SECRET_MOSCOW_PLACES = /,'').replace(/;\s*$/,'')); console.log(p.length, [...new Set(p.map(x=>x.city))])"
git diff --check
git status --short --branch
```

## Git-процесс

Обычно работаем прямо в `main`, если пользователь не просит отдельную ветку.

После законченной правки:

```powershell
git add ...
git commit -m "..."
git push origin main
```

Перед финальным ответом проверить `git status --short --branch`.
